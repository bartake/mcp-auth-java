How It Works

Technical documentation for the MCP Auth flow — user login through JWT verification, context propagation, and downstream API calls.

Overview

The example demonstrates a complete auth flow for MCP servers:

User login → OAuth (JWT issued)
       ↓
Agent receives JWT
       ↓
JWT passed to MCP server (Authorization: Bearer <token>)
       ↓
MCP verifies JWT + extracts roles
       ↓
Context attached to tool calls (AsyncLocalStorage)
       ↓
Downstream APIs use scoped tokens (same JWT forwarded)
Key design decisions:
  • Stateless JWT — No server-side sessions
  • Same token, multiple services — Auth issues; MCP and downstream both validate
  • AsyncLocalStorage — Propagates user context into async tool handlers
  • Role-based access — Downstream API enforces permissions via roles

Authentication Flow

Step 1: User Login

POST /login with {"username","password"} → returns JWT with sub, roles, scope.

Step 2: Agent Stores JWT

No session; token is kept client-side for subsequent requests.

Step 3: JWT Passed to MCP

Every MCP request includes Authorization: Bearer <token>.

Step 4: MCP Verifies and Extracts Roles

Middleware verifies signature, checks audience, parses sub, roles, scope into req.user.

Step 5: Context Attached to Tool Calls

Handler wraps execution in runWithUser(userCtx, fn) — sets AsyncLocalStorage. Tools call getUserContext() to access user.

Step 6: Downstream APIs Use Scoped Tokens

Tools forward the same JWT in Authorization; downstream API validates and checks roles (reader vs admin).

Components

Auth Service

Validates credentials, signs JWT with sub, roles, scope. Returns { access_token, token_type, expires_in }.

MCP Server

jwtAuth middleware → runWithUser → route JSON-RPC → tool handlers call getUserContext() to get user and _token for downstream calls.

User Context (AsyncLocalStorage)

Propagates user from HTTP request into async tool execution. Tool handlers read getUserContext() — same context set by runWithUser.

Downstream API

Validates JWT, enforces RBAC: GET /data (reader+), POST /data (admin only).
